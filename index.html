AI Threat Engine

Eight threat classes. Real-time detection.

The Shadow Queen threat detector runs statistical analysis over your credential event stream — no ML model required. Pure math over your own data.

T1 · VELOCITY_SPIKE
Burst Event Flood
Too many credential events from one origin in a short window. Indicates automated credential stuffing or bulk replay injection.
HIGH
T2 · VERSION_ROLLBACK
Replay Attack
Incoming event version lower than current credential version. Classic replay attack attempting to restore a revoked or suspended credential.
HIGH
T3 · PROOF_CYCLING
Key Compromise
Same credential reissued with a different proof hash. Indicates key compromise or credential impersonation attempt.
CRITICAL
T4 · RAPID_STATE_CYCLE
State Oscillation
Credential toggling between states faster than policy allows — suspicious automated behavior pattern.
HIGH
T5 · ORPHAN_TRANSITION
Ghost Credential
State transition arriving for a credential that was never issued at this node. Indicates out-of-band credential injection.
MEDIUM
T6 · STALE_FLOOD
Noise Injection
High volume of stale/duplicate events from one source — designed to overwhelm conflict detection and obscure real attacks.
HIGH
T7 · CONFLICT_CLUSTER
Coordinated Attack
Multiple distinct conflict types on the same credential — the signature of a coordinated, multi-vector attack.
CRITICAL
T8 · UNVERIFIED_EVENT_RATE
Hash Tampering
High fraction of events failing cryptographic hash verification. Indicates active MITM or event log tampering.
CRITICAL
Architecture

How Shadow Queen works.

Event-sourced. Content-addressed. Conflict-aware. Every credential change is an immutable event chained into a tamper-evident ledger.

STEP 01 · ISSUE

Publish a credential event

Issue a credential. The event is SHA-256 hashed, stored in the local ledger, and queued for replication to all target offices.

STEP 02 · REPLICATE

Federated delivery

The Replicator delivers events to peer stores. Duplicates are detected and rejected. Failed deliveries are tracked with retry metadata.

STEP 03 · DETECT

Threat scan

The AI threat detector scans the event stream for all 8 threat classes. Risk scores 0.0–1.0. Severity: low / medium / high / critical.

STEP 04 · AUDIT

Convergence proof

Run a convergence audit across all peers. Drift is flagged per credential. Export a signed evidence bundle (ZIP) for compliance.

# Issue a credential and replicate to two nodes
python -m shadowqueen.cli --db north.db --office north issue C-1 user:alice \
  --type license --proof-hash sha256:abc123 --targets south,east

# Run AI threat detection
python -m shadowqueen.cli --db north.db --office north detect
